Git: desarrollo colaborativo


Areas de GIT

Working directory

El área de trabajo corresponde a la tarjeta donde inicializamos el repositorio de GIT. Donde rara vez a estar utilizando la línea de comandos y es básicamente donde vamos a crear, editar y eliminar los diferentes archivos de nuestro proyecto

  • Dir || Ls

    • Éstos comandos muestran el contenido de la carpeta actual

  • Cd y path

    • Este comando cambia la carpeta actual por la seleccionada, para ir para atrás es "cd .."

  • mkdir

    • "mkdir carpeta" crea una carpeta con el nombre especificado "carpeta"

  • git init

    • Inicializa un repositorio de git en la carpeta actual

  • rm -rf ó rm -r -f

    • Elimina de manera forzada un repositorio


Staging area (index)

El area de control de cambios es aquella donde agregamos los archivos a los que vamos a estar haciéndole seguimiento, cada vez que agregamos un elemento al INDEX, se realiza una captura de código (snapshot) que luego deberá ser confirmada para determinar su permanencia en el historial de cambios

  • git status

    • Muestra el estado de los archivos del working directory con respecto l index, dentro de los estados más comunes están:

    • untrack:No se está haciendo seguimiento del archivo, irrecuperable en caso de pérdida
    • stage:Se realizó la captura del código, pero todavía no se han confirmado los cambios
    • commited:Los cambios fueron confirmados y se encuentran en el historial de versiones
    • modified:El archivo difiere con respecto a lo registrado en el index o la última confirmación (commit)
  • git add <file
    • pagrega al index el conjunto de archivos seleccionados
  • git diff compara los archivos del working directory con lo que está registrado en el index
  • git restore --staged <file> elimina la última captura del index sin modificar el archivo
  • git rm --cached se elimina el archivo del index, no se realiza seguimiento


Repositository (local/remote)

El es área donde se almacenan todas las configuraciones de cambios en archivos de tipo BLOB (binary large object file) distribuídos en una estructura de carpetas identificadas de manera hexadecimal

  • git commit: abre el editor predeterminado para realizar la configuración de cambios
  • git log: muestra le historial de cambios ordenados de manera descendente
  • git blame <file>: muestra los autores y confirmaciones de cada línea de archivo
  • git push <remote> <branch>: agrega los cambios de la rama al repositorio remoto
  • git fetch <remote>:captura el historial de cambios de nuestro repositorio remoto

Apuntadores de GIT

HEAD

Head es un apuntador movil que indica en qué punto del historial se encuentra, normalmente suele apuntar a la última confirmación que se ha hecho, pero, podemos utilizarlo para desplazarnos dentro del historial del cambios. También se utiliza como referente para otros comandos, como: checkout, diff, etc... si escribimos HEAD~n ó HEAD^^ estamos indiciando que queremos visualizar o comparar el commit que se encuentra n veces lo indicado

BRANCH

El branch es un apuntador dinámico que siempre se corresponde con el último commit de la historia. Se utiliza para facilitar la navegación dentro del historial de confirmaciones, y, también para probar características, corregir errores o realizar revisiones sin necesidad de afectar la historia principal
